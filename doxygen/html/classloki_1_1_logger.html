Detailed Description

Thread-safe singleton logger that writes to a file and stdout.

Initialisation

Call init() once at application startup before any log calls:

namespace fs = std::filesystem;
Logger::instance().init(fs::path("/workspace/OUTPUT/LOG"), "loki_homogeneity");
// Creates: /workspace/OUTPUT/LOG/loki_homogeneity_20260313_142301.log
loki::Logger::instance
static ILogger & instance()

Usage via macros (recommended)

LOKI_INFO("Opened file: " + path.string());
LOKI_WARNING("Line 42: unexpected column count, skipping.");
LOKI_ERROR("File not found: " + path.string());

Test injection

Replace the active instance with a mock before running tests:

Logger::setInstance(std::make_unique<MockLogger>());
loki::Logger::setInstance

Opens the log file and sets the minimum log level.

The log file name is constructed as: <logDir>/<prefix>_<YYYYMMDD_HHMMSS>.log

The log directory is created if it does not exist. If init() is not called, messages are written only to stdout.

Parameters
logDirDirectory where the log file will be created.
prefixFile name prefix (e.g. "loki_homogeneity", "test_loader").
minLevelMinimum severity to record; default INFO.
Exceptions

Static convenience accessor — initialises the default Logger singleton.
